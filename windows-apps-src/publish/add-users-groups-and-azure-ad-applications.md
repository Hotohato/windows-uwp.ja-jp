# <a name="add-users-groups-and-azure-ad-applications-to-your-partner-center-account"></a>ユーザー、グループ、およびパートナー センター アカウントに Azure AD アプリケーションを追加します。

**ユーザー**の[パートナー センター](https://partner.microsoft.com/dashboard) (**アカウント設定**)、パートナー センター アカウントにユーザーを追加する Azure Active Directory を使用することができます。 各ユーザーには、アカウントへのアクセスを定義するロール (またはカスタムのアクセス許可のセット) が割り当てられます。 追加することも[ユーザーのグループ](#groups)と[の Azure AD アプリケーション](#azure-ad-applications)パートナー センター アカウントへのアクセスを許可します。

ユーザーをアカウントに追加したら、[アカウントの詳細の編集](#edit)、[ロールやアクセス許可](set-custom-permissions-for-account-users.md)の変更、[ユーザーの削除](#remove)を行うことができます。

> [!IMPORTANT]
> 自分のアカウントにユーザーを追加するにはまず[、パートナー センター アカウントを組織の Azure Active Directory テナントに関連付ける](associate-azure-ad-with-partner-center.md)します。 

ユーザーを追加する場合は、割り当てることで、パートナー センター アカウントへのアクセスを指定する必要があります、[ロールまたはカスタムのアクセス許可のセット](set-custom-permissions-for-account-users.md)します。 

パートナー センターのすべてのユーザー (グループと Azure AD アプリケーションを含む) でのアクティブなアカウントがありますに注意してください[、パートナー センター アカウントに関連付けられている Azure AD テナント](associate-azure-ad-with-partner-center.md)します。 ユーザーの管理は、1 テナントずつ行います。ユーザーを追加または編集するテナントの管理者アカウントでサインインする必要があります。 パートナー センターで新しいユーザーの作成、Azure AD テナントにサインインして、組織の Azure AD テナントで同じ変更を加えますがパートナー センターでユーザーの名前に変更を加えるそのユーザーのアカウントは作成もできます。

> [!NOTE]
> 組織で使用する場合[ディレクトリ統合](https://go.microsoft.com/fwlink/p/?LinkID=724033)オンプレミスのディレクトリ サービスを Azure ad を同期することはできませんをパートナー センターで新しいユーザー、グループ、または Azure AD アプリケーションを作成します。 (または、オンプレミスのディレクトリ内の別の管理者) は、パートナー センターでそれらを追加できます前に、オンプレミスのディレクトリに直接作成する必要があります。

### <a name="create-a-new-azure-ad-application-account-in-your-organizations-directory-and-add-it-to-your-partner-center-account"></a>新しい Azure AD アプリケーションが、組織のディレクトリ内のアカウントし、パートナー センター アカウントへの追加

ブランド新しい Azure AD アプリケーションのアカウントにパートナー センターのアクセス権を付与する場合で 1 つを作成、**ユーザー**セクション。 パートナー センター アカウントでだけでなく、組織のディレクトリで新しいアカウントが作成されますこのことに注意してください。

> [!TIP]
> 任意の有効なアドレスを入力するには、パートナー センターの認証のためこの Azure AD アプリケーションは主を使用してユーザーに直接アクセスを必要はありません、**応答 URL**と**アプリ ID URI**限り、これらの値として、ディレクトリ内の他の Azure AD アプリケーションによって使用されません。

1.  **ユーザー**ページ (**アカウント設定**) を選択します**Azure AD アプリケーションを追加**します。
2.  次のページで次のように選択します。**新規 Azure AD アプリケーション**します。
3.  新しい Azure AD アプリケーションの **[応答 URL]** を入力します。 これは、ユーザーがサインインして、Azure AD アプリケーションを使うことができるようになる URL です ("アプリ URL" や "サインオン URL" と呼ばれる場合もあります)。 **[応答 URL]** は 256 文字以内にする必要があり、ディレクトリ内で一意でなければなりません。
4.  新しい Azure AD アプリケーションの **[アプリ ID URI]** を入力します。 これは、Azure AD アプリケーションの論理識別子であり、シングル サインオン要求を Azure AD に送るときに提供されます。 **[アプリ ID URI]** は、ディレクトリ内の各 Azure AD アプリケーションで一意の値する必要があります。またこの URI は、256 文字までにする必要があります。 **アプリ ID URI** について詳しくは、「[Azure Active Directory とアプリケーションの統合](https://docs.microsoft.com/azure/active-directory/develop/active-directory-integrating-applications#changing-the-application-registration-to-support-multi-tenant)」をご覧ください。
5.  **[ロール]** セクションで、Azure AD アプリケーションの[ロールまたはカスタムのアクセス許可](set-custom-permissions-for-account-users.md)を指定します。
6.  **[保存]** をクリックします。

Azure AD アプリケーションを追加または作成した後、**[ユーザー]** セクションに戻ってアプリケーションの名前を選択すると、テナント ID、クライアント ID、応答 URL、アプリ ID URI など、アプリケーションの設定を確認できます。

> [!NOTE]
> [Microsoft Store サービス](../monetize/using-windows-store-services.md)が提供する REST API を使用する場合は、サービスへの呼び出しの認証に使用する Azure AD アクセス トークンを取得するために、このページに表示されているテナント ID とクライアント ID の値が必要になります。

### <a name="manage-keys-for-an-azure-ad-application"></a>Azure AD アプリケーションのキーを管理する

Azure AD アプリケーションが Microsoft Azure AD でデータの読み取りや書き込みを行う場合は、キーが必要になります。 Azure AD アプリケーションのキーを作成するには、パートナー センターでその情報を編集します。 また、不要になったキーを削除することもできます。

1.  **ユーザー**ページ (**アカウント設定**)、Azure AD アプリケーションの名前を選択します。
    > [!TIP]
    > Azure AD アプリケーションの名前をクリックすると、キーが作成されたを使用して有効期限が切れる日付を含め、Azure AD アプリケーションのアクティブなキーのすべてが表示されます。 不要になったキーを削除するには、**[削除]** をクリックします。

2.  新しいキーを追加するには、次のように選択します。**新しいキーの追加**します。
3.  **[クライアント ID]** と **[キー]** の値を示す画面が表示されます。
    > [!IMPORTANT]
    > 印刷するか、この情報をコピーすることを確認してこのページから移動した後に再度アクセスすることはできません。

4.  以上のキーを作成する場合は、選択**別のキーを追加**します。

## <a name="edit-a-user-group-or-azure-ad-application"></a>ユーザー、グループ、または Azure AD アプリケーションの編集

ユーザー、グループ、またはパートナー センター アカウントに Azure AD アプリケーションを追加した後、アカウント情報への変更を行うことができます。 

> [!IMPORTANT]
> 加えられた変更[役割またはアクセス許可](set-custom-permissions-for-account-users.md)は、パートナー センターへのアクセスにのみ影響します。 (Azure AD アプリケーションのユーザーの名またはグループのメンバーシップ、または応答 URL とアプリ ID URI の変更) などの他のすべての変更は、組織の Azure AD テナントにも、パートナー センター アカウントと反映されます。 

1.  **ユーザー**ページ (**アカウント設定**)、ユーザー、グループ、または編集する Azure AD アプリケーションのアカウントの名前を選択します。
2.  必要な変更を行います。 編集できる項目は次のとおりです。
    -   **ユーザー**の場合は、ユーザーの姓、名、またはユーザー名を編集できます。 また、**[グループ メンバーシップ]** セクションで、グループを選ぶまたは選択を解除して、ユーザーのグループ メンバーシップを更新することもできます。
    -   **グループ**の場合は、グループの名前を編集できます。 (グループ メンバーシップを更新するには、グループに追加またはグループから削除するユーザーを編集し、**[グループ メンバーシップ]** セクションを変更します。)
    -   **Azure AD アプリケーション**の場合は、**[応答 URL]** または **[アプリケーション ID/URI]** に新しい値を入力できます。
    組織のディレクトリにも、パートナー センター アカウントでこれらの変更が行われることに注意してください。
3.  パートナー センターのアクセスに関連する変更は、次のように選択します。 または、適用、または選択するロールの選択を解除**アクセス許可のカスタマイズ**必要な変更を行います。 これらの変更にのみ影響をパートナー センターにアクセスし、組織の Azure AD テナント内での権限は変更されません。
3.  **[保存]** をクリックします。

## <a name="remove-users-groups-and-azure-ad-applications"></a>ユーザー、グループ、および Azure AD アプリケーションの削除

パートナー センター アカウントからユーザー、グループ、または Azure AD アプリケーションを削除するには、選択、**削除**上の名前で表示されるリンク、**ユーザー**ページ。 を削除することが確定した後、ユーザー、グループ、または Azure AD アプリケーションできなく (追加しない限り、後でもう一度)、パートナー センター アカウントにアクセスできません。

> [!IMPORTANT]
> ユーザー、グループ、または Azure AD アプリケーションを削除すると、パートナー センター アカウントへのアクセスがあるが不要になったことを意味します。 この操作により、組織のディレクトリからユーザー、グループ、または Azure AD アプリケーションが削除されることは**ありません**。
